# IMAP Mail Migration

Скрипт переносит почту с Яндекс 360 на свой IMAP-сервер. Умеет докачивать с места остановки, переподключаться при обрывах и при временной потере сети (например после отключения VPN), сохраняет стандартные флаги и даты писем. Подходит для длительных миграций с большими ящиками.

## Что понадобится

- Python 3.10+
- Пароль приложения Яндекса (не основной пароль): [id.yandex.ru/security/app-passwords](https://id.yandex.ru/security/app-passwords) — создать для «Почта».
- Учётные данные целевого IMAP-сервера.

## Настройка

Конфиг с паролями в репозиторий не коммитится. Сначала создайте свой конфиг из примера:

```bash
cd imap_migrate
cp config.yaml.example config.yaml
nano config.yaml   # или любой редактор
```

В `config.yaml` обязательно заполните:

- **source:** `user` и `password` — ящик Яндекса и пароль приложения.
- **destination:** `host`, `user`, `password` — хост и учётка на вашем IMAP-сервере.

Остальные параметры в `options` можно оставить по умолчанию. Там есть таймауты, паузы между письмами, размер батча и т.д. — их имеет смысл трогать, только если появятся таймауты или лимиты со стороны Яндекса.

## Запуск

Удобнее всего через `run.sh`: он поднимет venv и подставит нужный конфиг.

```bash
chmod +x run.sh
./run.sh
```

При первом запуске создаётся виртуальное окружение и ставятся зависимости. Дальше:

```bash
# Посмотреть папки на источнике
./run.sh list-folders

# Пробный прогон без записи
./run.sh dry-run

# Сам перенос
./run.sh run

# Перенос и потом сверка количества писем по папкам
./run.sh verify
```

По умолчанию используется `config.yaml` из текущей папки. Если конфиг лежит в другом месте или у вас несколько конфигов:

```bash
IMAP_CONFIG=/path/to/config.yaml ./run.sh run
```

Только отдельные папки:

```bash
./run.sh run --folders INBOX,Sent
```

## Повторный запуск и обрывы

Состояние пишется в `migration_state.json`. Уже перенесённые письма (по Message-ID) при следующем запуске пропускаются. Если скрипт упал или вы остановили его по Ctrl+C — просто запустите `./run.sh run` снова: он продолжит с того же места.

При потере сети (VPN выключили, пропал интернет) скрипт будет пытаться переподключиться с увеличивающейся паузой (до значения `reconnect_max_wait` в конфиге). После восстановления связи можно снова запустить миграцию — лишнего копирования не будет.

## Ручной запуск без run.sh

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

python imap_migrate.py -c config.yaml --list-folders
python imap_migrate.py -c config.yaml --dry-run
python imap_migrate.py -c config.yaml
```

## Параметры командной строки

| Параметр | Описание |
|----------|----------|
| `-c`, `--config` | Путь к конфигу (обязательно) |
| `-n`, `--dry-run` | Проверка без записи на целевой сервер |
| `-f`, `--folders` | Папки через запятую (иначе обрабатываются все) |
| `-v`, `--verbose` | Подробный лог (DEBUG) |
| `--verify` | После переноса сравнить число писем по папкам |
| `--list-folders` | Показать папки источника и выйти |

## Что лежит в проекте

- `imap_migrate.py` — основной скрипт
- `config.yaml.example` — пример конфига (скопируйте в `config.yaml` и отредактируйте)
- `config.yaml` — ваш конфиг с паролями (в git не попадает)
- `run.sh` — создание venv и запуск команд
- `requirements.txt` — зависимости (pyyaml, tqdm)
- При работе создаются: `.venv/`, `migration_state.json`, `migration.log`

## Частые проблемы

**Ошибка входа (Login failed)** — используйте именно пароль приложения Яндекса и проверьте, что в почте включён доступ по IMAP.

**Обрывы соединения, таймауты** — можно увеличить `timeout` и `throttle` в конфиге; при нестабильной сети скрипт сам переподключается, после появления сети запустите его снова.

**В веб-интерфейсе у писем лишние теги** — нестандартные флаги вроде `$NotJunk` при переносе отбрасываются; если что-то всё равно появляется, в конфиге есть `exclude_flags`.

**Python не находится** — на macOS: `brew install python@3.12` или установка с python.org.

**Число писем не сходится при verify** — часто помогает повторный запуск `./run.sh run`, если в первый раз были единичные ошибки или обрывы.
